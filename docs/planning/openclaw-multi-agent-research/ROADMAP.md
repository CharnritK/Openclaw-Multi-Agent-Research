# ROADMAP

## Objective

Produce a practical, continuously updated research program for an OpenClaw-centered multi-agent stack with explicit Paperclip and Hermes integration tracks.

## Approach

1. Start with Point's real operating model, not generic framework marketing.
2. Compare candidate frameworks against bounded orchestration needs.
3. Evaluate Paperclip and Hermes as integration layers, not automatic core dependencies.
4. Turn research into a decision-ready architecture recommendation.
5. Harden medium-confidence recommendations into explicit triggers, contracts, and evidence maps before live prototype pressure.
6. Advance in small, commit-sized tasks.

## Phases

### Phase 1, planning and baseline
- define objective, scope, assumptions, risks, and research workflow
- create canonical planning bundle
- establish evaluation dimensions and active tasks
- exit criteria: planning bundle exists and next research tasks are ready

### Phase 2, framework landscape scan
- evaluate a small set of candidate frameworks and agent operating patterns
- compare fit for OpenClaw-style control-plane-first usage
- exit criteria: framework comparison note and shortlist completed

### Phase 3, Paperclip integration track
- identify where Paperclip adds value: budgets, goals, org chart, heartbeats, dashboard
- assess complexity, overlap, and integration boundaries with OpenClaw
- exit criteria: Paperclip recommendation written with best-fit and no-fit cases

### Phase 4, Hermes integration track
- identify viable Hermes integration models: side-by-side, bridge, shared channel, or specialist runtime
- assess risk around duplicated gateways, state drift, and operator complexity
- exit criteria: Hermes recommendation written with integration boundary and risks

### Phase 5, recommended target architecture
- synthesize findings into one recommended architecture and one fallback
- define proof-of-concept scope and validation plan
- exit criteria: decision-ready architecture and prototype checklist completed

### Phase 6, continuous research loop
- keep tracking new repos, docs, and implementation examples
- update decisions, risks, and task readiness as evidence improves
- exit criteria: ongoing maintenance mode with a clear next-best task always visible

### Phase 7, decision hardening
- convert the current architecture recommendation into a traceable evidence-backed checkpoint
- define one Paperclip adoption-trigger workflow and one Hermes specialist contract that can reopen research without broadening scope
- centralize major claims, source classes, counterevidence, and reopen triggers in one evidence map
- rerun a short architecture review after those artifacts land
- exit criteria: the recommendation is either reaffirmed or narrowed with explicit reopen rules and no broad framework expansion
